# 🤖 Agent Context: Personal Finance PWA

This document serves as a high-level knowledge base for AI agents interacting with the Personal Finance PWA codebase. It aims to provide essential context, architectural patterns, and data schemas to minimize token usage and improve code generation accuracy.

## 🌟 Project Overview
A secure, offline-first Progressive Web Application for comprehensive personal finance management.
- **Core Mission**: Privacy-centric financial tracking with AI-powered insights and smart cloud backups.
- **Tech Stack**: React 19, TypeScript, Vite 7, Bootstrap 5, Dexie.js (IndexedDB).
- **Hosting**: Netlify (with Netlify Functions).

## 🏗️ Architecture & Folder Structure
- `src/app/`: Root app configuration, routes, providers (`App.tsx`, `main.tsx`).
- `src/domains/`: Modular slices representing functional core business domain areas:
  - `auth/`: Biometric authenticator and user lock state hooks (`bioLockContext.tsx`).
  - `backups/`: Google Drive backup synchronizations and logs.
  - `analytics/`: Dashboards, visual graphs, and dynamic financial projection calculations.
- `src/infrastructure/`:
  - `db/db.ts`: Dexie database instance.
  - `db/dbMigrations.ts`: Core schema version upgrades and index migrations.
- `src/service-worker/`: Custom precaching and client updates Service Worker (`sw.ts`).
- `src/shared/`: Cross-cutting layout controls, daily tip cards, logging services, financial formulas (`financialUtils.ts`), and undo-redo stacks.

## 💾 Data Schema (Dexie.js)
The app uses IndexedDB via Dexie. Key tables and their purposes:
- `configs`: Encrypted local settings (API keys, themes).
- `assetsHoldings`: Current investments, SIPs, and allocations.
- `liabilities`: Loans, start dates, and durations.
- `upcomingExpenses`: Future financial obligations with completion status.
- `insurances` & `insuranceTypes`: Insurance tracking module data (Premium, Sum Assured, Dates).
- `income` & `cashFlow`: Monthly/Yearly cash movement.
- `goals`: Financial targets linked to asset purposes.
- `assetsProjection` & `liabilitiesProjection`: User-defined future financial events.

**Important Note**: All data is stored locally. Syncing to Google Drive is a manual or triggered backup process, not a real-time database sync.

## 🤖 AI Implementation (Google Gemini)
- **Model**: Primarily uses Gemini Flash/Pro models.
- **Access**: Direct client-to-API communication using the user's provided API key.
- **Permissions**: Read-only access to the local database to provide insights (Net worth analysis, spending trends).
- **Features**: Natural language queries, image rendering in chat.

## 🛠️ Key Business Logic
- **FI Levels**: Calculates multiple milestones including Lean FIRE (4% SWP), Financial Independence, and Abundant Wealth. Focuses on retirement-specific assets.
- **Undo/Redo**: Managed via `historyService.ts` using Dexie hooks to track every DB operation.
- **Gold/Silver Rates**: Fetched from GoldAPI and cached daily in local storage, includes international-to-Indian rate conversion.

## 📜 Development Guidelines
- **Styling**: Use `react-bootstrap` components combined with custom SCSS in `src/styles`.
- **Database**: Always use the `db` instance from `@/infrastructure/db/db`. Use `useLiveQuery` from `dexie-react-hooks` for reactive UI.
- **Naming**: Use camelCase for variables/functions and PascalCase for components/interfaces.
- **Commits**: Follow conventional commits (feat, fix, refactor, chore).
